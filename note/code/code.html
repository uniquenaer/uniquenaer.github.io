# 从三元表达式说起

坚持使用最直观的编码方式，而不是追求代码简短，真的可以避免很多不必要的错误

在看《极客时间》的代码精进之路之前，我写代码是追求代码简短，认为用少的行数写出同样功能的代码是件很有成就感的事情，却忽略了代码的可读性。

# 什么是优秀的代码?

优秀的代码的特点是——经济、规范、安全 经济指的是较少的人力、物力、财力、时间、空间，来获得较大的成果获收益。 规范：程序员间合作维护交流方便 安全：避免一些不容忽视的安全问题，对公司造成不容忽视的恶劣影响。

极客时间版权所有: https://time.geekbang.org/column/article/75980

这需要工程师脱离研发的角度，站在一个全局角度理解。

# 该如何理解经济呢？

这需要放到软件的整个生命周期理解.一个典型的软件生命周期，大致可以划分计划、分析和设计、代码实现、测试、运营和维护六个阶段。

极客时间版权所有: https://time.geekbang.org/column/article/75980

不同的公司选择的软件开发流程不一样，不同的时期流程也不一样，这就需要我们结合当前显示环境思考，写出最适合当前现实环境的代码。
