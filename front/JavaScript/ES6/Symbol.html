Symbol - ECMAScript 6入门

# Symbol是什么？

  • ES6 新引入的，目的是为解决对象中自定义属性名时易产生冲突的问题，保证每个属性的名字都是独一无二

  • Symbol通过Symbol函数生成，

  • Symbol函数前不能用new 来生成，会报错；因为Symbol函数的结果是一个值类型，不是对象

# Symbol 函数

  • 接受一个字符串作为参数，标识对symbol的描述，如传入的是对象，则会转成字符串。即便传入的描述是一样的，最后的两个symbol也是不相等的。

  • Symbol 值不能与其他类型的值进行运算，会报错

Symbol 不能隐式的转为字符串，可以显式的通过toString的方式转化为字符串

  • Symbol 值也可以转为布尔值，但是不能转为数值

  • 每执行一次Symbol都会生成新的Symbol值，及时每次传入的参数都一样。

调用Symbol("cat")30 次，会返回 30 个不同的 Symbol 值。

# Symbol作为对象属性名使用

1 Symbol 值作为对象属性名时，不能用点运算符。

2 Symbol 值作为属性名时，该属性还是公开属性，不是私有属性。


let mySymbol = Symbol();

// 第一种写法
let a = {};
a[mySymbol] = 'Hello!';

// 第二种写法
let a = {
  [mySymbol]: 'Hello!'
};

// 第三种写法
let a = {};
Object.defineProperty(a, mySymbol, { value: 'Hello!' });

// 以上写法都得到同样结果
a[mySymbol] // "Hello!"

# 属性名的遍历

  • Symbol 作为属性名，该属性不会出现在for...in、for...of循环中，也不会被Object.keys()、Object.getOwnPropertyNames()、JSON.stringify()返回。

  • 它也不是私有属性，

  • 通过Object.getOwnPropertySymbols方法，可以获取指定对象的所有 Symbol 属性名。

  • Reflect.ownKeys方法可以返回所有类型的键名，包括常规键名和 Symbol 键名。

# Symbol.for()，Symbol.keyFor()

1 Symbol.for

Symbol.for 接受一个字符串作为参数，搜索有没有以该参数作为名称的 Symbol 值。如果有，就返回这个 Symbol 值，否则就新建并返回一个以该字符串为名称的 Symbol 值,如果你调用Symbol.for("cat")30 次，每次都会返回同一个 Symbol 值

2 Symbol.keyFor

返回一个已登记的 Symbol 类型值的key。

Symbol.for返回的Symbol会在全局登记，Symbol函数返回的Symbol不会再全局登记
