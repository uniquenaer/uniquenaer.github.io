# 消息推送（sse）

# 安装 EventSource

EventSource 只支持现在浏览器，故需补丁兼容 IE10，event-source-polyfill 很好的兼容了其它浏览器，更扩展了标准中的 EventSource 的 API，使其更好用。具体使用参考EventSource。

npm install event-source-polyfill

# 使用

  1. 获得当前用户的 events 权限，即得到当前用户可监听的推送信息类别。请求/user/information得到：
{
  "active": 1,
  "email": "1312342604@qq.com",
  "events": ["名称", "激活"],
  "group_id": null,
  "id": 1,
  "nickname": "pedro",
  "super": 2
}

events字段代表当前用户可监听的 Events。

  1. 连接。CMS 对每个接口都有权限认证的机制，所以连接时必须带上 token。

  1. 监听。给eventSource增加监听事件：
eventSource.addEventListener(
  '名称',
  function(e) {
    console.log(e);
    if (e.data !== 'None') {
      pre.innerHTML = e.data;
    }
    targetContainer.innerHTML = e.data;
  },
  false
);

上述代码便可监听名称这个 event 的推送信息。前端可以使用工厂方法为每个event添加监听。
